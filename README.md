# LLM-ассистент для абитуриентов

Репозиторий содержит учебный MVP ассистента для абитуриентов, который отвечает на вопросы о поступлении, образовательных программах, стоимости обучения, проходных баллах, количестве мест, требованиях к ЕГЭ и устройстве мегакластеров.

В основе проекта — RAG-подход с разделением источников по типам данных: система сначала определяет тип запроса, затем ищет ответ в подходящей базе и только после этого формирует итоговый ответ. Для вопросов, где важны точные значения, используются структурированные таблицы, чтобы не допускать выдуманных чисел.

## Что делает проект

Система умеет:
- принимать вопрос на естественном языке;
- фильтровать пустые, слишком длинные и токсичные сообщения;
- определять тип запроса;
- искать ответ в FAQ, текстовой базе знаний или таблице программ;
- извлекать точные значения по стоимости, баллам, местам и ЕГЭ;
- возвращать краткий и безопасный ответ без галлюцинаций;
- работать через API и Telegram-бота.

## Состав репозитория

### `main.py`
Основное ядро проекта. Здесь собрана вся логика обработки запроса: модерация, классификация вопроса, retrieval по нескольким источникам, извлечение точных данных и сборка финального ответа.

### `api_server.py`
FastAPI-сервер, который поднимает HTTP API для общения с моделью. Через него можно отправлять обычный текстовый запрос или `.docx`-файл и получать готовый ответ.

### `tg_bot.py`
Telegram-бот, который выступает пользовательским интерфейсом. Он принимает сообщения и документы, отправляет их в API и возвращает ответ пользователю.

### `experiments.ipynb`
Ноутбук с экспериментами и разработкой пайплайна. В нём собраны промежуточные проверки, тестирование retrieval, логики маршрутизации, обработки вопросов и прототипирования ответов.

### `requirements.txt`
Список библиотек, необходимых для запуска проекта: API, Telegram-интерфейс, обработка таблиц и документов, retrieval, эмбеддинги и модели. fileciteturn0file0

## Папка `data/`

В папке `data/` лежат основные источники знаний проекта:
- `all_program.xlsx` — структурированная таблица по образовательным программам: названия, мегакластеры, институты, направления, треки, длительность, стоимость, проходные баллы, количество мест, ЕГЭ и описания;
- `Database.xlsx` — FAQ-база с типовыми вопросами и готовыми ответами;
- `Database-2.xlsx` — текстовая база знаний для общих вопросов о мегакластерах, направлениях и устройстве обучения;
- `ru_abusive_words.txt` и `ru_curse_words.txt` — словари для базовой модерации токсичных запросов;
- `user_questions.xlsx` — набор пользовательских вопросов для тестов, отладки и оценки качества.

## Как это работает

Общий пайплайн выглядит так:
1. Пользователь отправляет вопрос.
2. Система проводит модерацию.
3. Определяется тип запроса: FAQ, общий текстовый вопрос или вопрос к табличным данным.
4. Выбирается соответствующий источник.
5. Выполняется retrieval или точечный поиск значения.
6. Пользователь получает итоговый ответ либо честный fallback, если данных недостаточно.

## Краткая сводка

Это репозиторий с уже собранным демонстрационным ассистентом для приёмной кампании. В нём есть:
- данные;
- ядро обработки вопросов;
- API;
- Telegram-бот;
- ноутбук с экспериментами.

То есть репозиторий представляет собой не набор разрозненных файлов, а цельный MVP: от базы знаний и логики retrieval до интерфейса, через который систему можно показать на защите.

## Запуск

```bash
pip install -r requirements.txt
uvicorn api_server:app --reload
python tg_bot.py
```

После этого можно тестировать API напрямую или подключать Telegram-бота.
